Comandos git:

git init

O comando "git init" cria um repositório git na pasta que você quer enviar para o Github. Ou seja, a pasta será um projeto gerenciado pelo git.

git add .

O comando "git add ." vai enviar os arquivos da pasta para uma área temporária chamada "stage".

git status

O comando "git status" mostra as modificações feitas para ser "comitadas", ou seja, para serem salvas

git commit -m "mensagem explicativa"

O comando "git commit -m "mensagem explicativa" é o comando que salva uma nova versão no seu repositório git. E junto você envia uma mensagem explicando as alterações que você fez.

git branch -M main

O comando "git branch -M main" serve para garantir que o github vai usar o nome main para salvar o histórico de commits. Antigamente era usado master.

git remote add origin git@github.com:seuusuario/seurepositorio.git

O comando "git remote add origin git@github.com:seuusuario/seurepositorio.git" faz a associação do seu projeto que está no seu computador, com o seu projeto que você criou lá no Github.

git push -u origin main

O comando "git push -u origin main" envia pela primeira vez um projeto que você ainda não enviou para o Github antes.
